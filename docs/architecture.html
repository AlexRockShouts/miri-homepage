🏗️ Architecture Overview

Miri is designed as a modular, autonomous system with a focus on local execution, persistent memory, and extensibility.

System Topology

  • 1.
    Client Layer: Web Dashboard (SvelteKit) and TypeScript SDK interacting via HTTP/WS.
  • 2.
    Gateway: Orchestrates the REST/WebSocket server (Gin), Multi-channel messaging (WhatsApp, IRC), and core Agent lifecycle.
  • 3.
    Engine (Eino): A graph-based ReAct engine that handles reasoning loops, tool calls, and checkpointing.
  • 4.
    The Brain: Asynchronous cognitive maintenance engine managing long-term memory (Vector DB) and reasoning quality (Mole-Syn).

The Brain: Cognitive Architecture

Miri's memory isn't just a list of messages; it's a dual-layered system powered by chromem-go.

  • Facts: Atomic pieces of knowledge and user preferences.
  • Summaries: Narrative records of conversation segments for continuity.
  • Mole-Syn: Advanced reasoning framework that models thoughts as interconnected "bonds" (Deep Reasoning, Self-Reflection, Exploration).

Multi-Channel Support

Miri can be controlled via various interfaces beyond the dashboard:

  • WhatsApp: Scan a QR code to interact with Miri via mobile.
  • IRC: Connect Miri to your favorite IRC channels.
  • CLI: Direct interaction via shell.

Persistence & Sandboxing

All state is stored locally in ~/.miri. File operations are strictly sandboxed in ~/.miri/generated to prevent unauthorized access to the host system.
