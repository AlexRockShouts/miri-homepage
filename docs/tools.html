🔧 Agent Tools

Miri uses a ReAct loop to execute tools and solve tasks autonomously.

Sandboxing

For security and organization, all tool-initiated file operations and script executions are automatically redirected to a sandboxed directory:

~/.miri/generated/

Available Tools

Tool Description Key Parameters
web_search Search the web via SearXNG/Google. query
web_fetch Download and parse content from a URL. url
execute_command Execute shell commands in the sandbox. command
file_manager List and share files from the sandbox. action ('list', 'share'), path, channel, device

Recurring Tasks

Miri features a built-in CronManager that allows the agent to schedule and execute tasks autonomously. These tasks are persisted and survive server restarts.

📅 Task Manager Tool

The task_manager tool is the primary interface for the agent to manage these schedules.

  • Add: Create a new task with a name, cron expression, and prompt.
  • List: View all currently scheduled tasks and their IDs.
  • Delete: Remove a task by its unique ID.

Reporting & Notifications

When a task executes, its results can be reported back to several targets:

  • Session: Results are injected back into the miri:main:agent session.
  • WebSocket: Active dashboard clients receive real-time push notifications of task completion.
  • Channels: Task results can be sent directly to WhatsApp or IRC.
