🚀 Getting Started with Miri

Prerequisites

  • Go: v1.25 or later (for Miri Server)
  • Node.js: v20 or later (for Miri Dashboard)
  • xAI API Key: Required for the agent's reasoning capabilities.

1. Miri Server Setup

The core engine is written in Go and provides the REST/WebSocket API.

# Clone the server repository
git clone https://github.com/AlexRockShouts/miri-main.git
cd miri-main

# Build the server
make build

# Run the server (default: http://localhost:8080)
./bin/miri-server

2. Miri Dashboard Setup

A modern SvelteKit interface for monitoring and interaction.

# Clone the dashboard repository
git clone https://github.com/AlexRockShouts/miri-dashboard.git
cd miri-dashboard

# Install dependencies
npm install

# Link local SDK (if necessary, see below)
# Update package.json to point to your local @miri/sdk path
# "dependencies": { "@miri/sdk": "link:../miri-main/api/sdk/typescript" }

# Configure environment
cp .env.example .env

# Start development server
make dev

Dashboard Configuration (.env)

Edit the .env file to connect the dashboard to your Miri Server:

Variable Description Example
PUBLIC_MIRI_SERVER_URL The base URL of your Miri Server. http://localhost:8080
MIRI_SERVER_KEY The server.key from your config.yaml. your-secret-key
MIRI_ADMIN_USER The server.admin_user from your config.yaml. admin
MIRI_ADMIN_PASSWORD The server.admin_pass from your config.yaml. admin-password

3. Configuration

Miri uses a config.yaml (usually in ~/.miri/) for server settings and model providers. It supports environment variable substitution.

storage_dir: ~/.miri

server:
  addr: :8080
  key: "your-server-key" # X-Server-Key for standard API
  admin_user: "admin"    # Basic Auth username for Admin API
  admin_pass: "secret"   # Basic Auth password for Admin API

models:
  mode: merge
  providers:
    xai:
      baseUrl: "https://api.x.ai/v1"
      apiKey: "$XAI_API_KEY"
      api: "openai"
      models:
        - id: "xai/grok-4-1-fast-reasoning"
          name: "grok-4-1-fast-reasoning"
          contextWindow: 2000000
    huggingface:
      baseUrl: "https://api-inference.huggingface.co/v1/"
      apiKey: "$HF_TOKEN"
      api: "openai"

agents:
  defaults:
    model:
      primary: "xai/grok-4-1-fast-reasoning"

4. Startup & Deployment

While you can run Miri manually, it is recommended to run it as a background service for continuous operation.

Linux (systemd)

Create a service file (e.g., /etc/systemd/system/miri@user.service) using the provided template:

sudo cp templates/systemd/miri.service /etc/systemd/system/miri@$(whoami).service
sudo systemctl daemon-reload
sudo systemctl enable miri@$(whoami)
sudo systemctl start miri@$(whoami)

macOS (launchd)

Copy the plist template to your LaunchAgents directory:

mkdir -p ~/Library/LaunchAgents
cp templates/launchd/com.miri.agent.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.miri.agent.plist

Windows (NSSM)

Use NSSM to install Miri as a Windows service:

nssm install Miri "C:\path\to\miri.exe"
nssm start Miri

Environment Variables

Variable Description Default
MIRI_STORAGE_DIR Base directory for all data (soul, brain, skills). ~/.miri
XAI_API_KEY Your xAI API key (if not set in config). -
PORT Override the server port. 8080

First Run: Bootstrapping

On its first run, Miri creates a "personality" and a toolset in your storage directory:

  • Soul: Copies templates/soul.md to ~/.miri/soul.md. You can edit this to change Miri's personality.
  • Skills: Copies core skills (learn, skill_creator) to ~/.miri/skills/.
  • Memory: Initializes a fresh vector database in ~/.miri/vector_db/.

Authentication Methods

  • Server Key: Pass via X-Server-Key header for all /api/v1/* endpoints.
  • Basic Auth: Use configured admin_user and admin_pass for all /api/admin/v1/* endpoints.
  • WebSocket: Use the miri-key sub-protocol or token query parameter.
