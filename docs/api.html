📡 API & SDK

Miri provides a comprehensive REST and WebSocket API for interaction and management.

Authentication

  • Server Key: Required for /api/v1/* and /ws. Pass via X-Server-Key header or WS sub-protocol.
  • Admin Auth: Basic Auth (configured in config.yaml) for /api/admin/v1/*.

Standard API (v1)

POST /api/v1/prompt

Send a prompt to the agent and get a blocking response.

curl -X POST http://localhost:8080/api/v1/prompt \
  -H 'Content-Type: application/json' \
  -H 'X-Server-Key: your-key' \
  -d '{"prompt": "Hello Miri!"}'

GET /api/v1/prompt/stream

Streaming response via Server-Sent Events (SSE).

WebSocket (/ws)

Full-duplex streaming for real-time interaction. Sub-protocol: miri-key, <key>.

TypeScript SDK

The SDK is the easiest way to integrate Miri into your applications. It's used by the Miri Dashboard.

import { Api } from '@miri/sdk';

const api = new Api({
  baseUrl: 'http://localhost:8080',
  securityWorker: (token) => ({
    headers: { 'X-Server-Key': token }
  })
});

const response = await api.prompt.createPrompt({
  prompt: "What is the capital of Switzerland?"
});

Admin API

  • GET /api/admin/v1/config: Current server configuration.
  • GET /api/admin/v1/skills: List installed skills.
  • GET /api/admin/v1/tasks: Monitor scheduled tasks.
